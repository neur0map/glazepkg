# Bugfixes: #16, #21, #22

## #16 — AUR packages showing as pacman in install search

Root cause: Pacman.Search() strips the repo prefix (aur/, core/, extra/) and
hardcodes SourcePacman for all results.

Fix: Check repo prefix before stripping. If "aur", set SourceAUR.

File: internal/manager/pacman.go — Search() method

## #21 — System Ruby gems polluting gem list

Root cause: gem list --local includes system gems marked with "default:" in
the version string. Code strips the marker but includes them anyway.

Fix: Skip gems with "default:" marker during scan.

File: internal/manager/gem.go — Scan() method

## #22 — Missing descriptions for managers

Add Describer interface to 7 managers:

- cargo — cargo info <pkg>
- bun — npm info <pkg> description (bun packages are npm packages)
- scoop — scoop info <pkg>
- conda — conda info <pkg>
- go — go doc or go list -m -json
- chocolatey — choco info <pkg>
- winget — winget show <pkg>

Cannot be fixed (no description source):
- nuget — filesystem-based, no query tool
- mas — mas info doesn't return descriptions
- windows_updates — API doesn't expose descriptions
